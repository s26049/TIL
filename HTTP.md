## HTTP  

### HTTP란 무엇인가
웹 브라우저(클라이언트)와 웹 서버가 HTML, 이미지 등 하이퍼텍스트 문서를 교환하기 위해 사용하는 통신 규약

### HTTP Method
클라이언트 - 서버 구조에서 요청과 응답이 이루어지는 방식을 의미

### 사용 이유
리소스와 동작을 분리하기 위함

### Method 종류
크게 8가지로 나뉜다.

- **GET : 리소스를 조회**
- **POST : 데이터를 추가, 등록**
- **PUT :  리소스 대체, 수정 / 해당 리소스가 없으면 새롭게 생성**
- **DELETE : 리소스 삭제**
- **PATCH : 리소스 부분 변경(수정)**
- HEAD : HTTP 메시지의 body 부분을 제외하고 리소스를 조회
- OPTIONS : 서버와 브라우저가 통신하기 위한 통신 옵션을 확인하기 위함
(서버가 어떤 method, header, content-type을 제공하는지 알 수 있음)
- CONNECT : 대상 자원으로 식별되는 서버에 대한 연결 요청

### 주로 사용하는 Method

**GET**
- 리소스를 **조회**하는 메서드
(ex. URL 입력, 링크 클릭)
- GET 요청의 **멱등성** : 여러번 조회 요청을 하여도 리소스는 변하지 않음
- 서버에 데이터를 전달하는 경우 쿼리스트링을 통하여 전달


**POST**
- 주로 새로운 리소스를 생성하는데 사용
- 성공적으로 생성을 완료하면 201 HTTP 응답을 반환
- 데이터를 메시지 바디에 쿼리 파라미터 형식(Key - Value)으로 전달
- GET 방식과 비교하였을 때 보안성에서 이점을 지님
- 데이터 전송 시 body에 담아 전달하여 입력 제한이 없음


**PUT**
- 리소스를 완전히 대체하는 개념(덮어쓰기)
- 클라이언트가 리소스를 식별할 수 있음
- 부분 수정이 불가능
- 멱등성을 지님


**PATCH**
- PUT과 같이 리소스를 수정하나 리소스를 **부분 변경** 한다는 점에서 차이가 있음
- 멱등성을 지니지 않음


### HTTP Status code
HTTP 요청이 성공적으로 완료되었는지 알려주며, 응답은 5개의 그룹으로 나뉜다.
- 정보 응답
- 성공 응답
- 리다이렉션 메시지
- 클라이언트 에러 응단
- 서버 에러 응답

### HTTP 무상태성
서버가 클라이언트의 상태를 보여주지 않는 특징


**DELETE**
- 리소스를 제거하는 역할
- 멱등성을 지님
